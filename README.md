#   End-to-End Retail Product Analysis with SQL | BigBasket Case Study
### Project Overview
BigBasket is one of India’s largest online grocery platforms, offering products across multiple categories and brands.
This project analyzes the retail product catalog snapshot to uncover insights related to category dominance, pricing strategy, discount behaviour, and customer ratings using SQL-driven analysis.

### Objectives
To analyze BigBasket product data using SQL in order to:
- Understand category & brand pricing behaviour
- Evaluate discount strategies
- Identify stock availability issues
- Generate actionable business insights

### Dataset information

* Source - [Kaggle](https://www.kaggle.com/datasets/surajjha101/bigbasket-entire-product-list-28k-datapoints)

Each row represents a unique SKU (Stock Keeping Unit) for a product. Duplicate product names exist because the same product may appear multiple times in different package sizes, weights, discounts, or categories to improve visibility – exactly how real catalog data looks.

Key Columns:

*  `sku_id ` - Unique identifier for each product entry
*  `product_name ` - Title of the product (as they're listed)
*  `category ` - Category into which product has been classified
*  `sub_category ` - Subcategory into which product has been kept
*  `brand ` - Brand of the product
*  `sale_price ` - Price at which product is being sold on the site
*  `market_price ` - Market price of the product
*  `product_type ` - Type into which product falls
*  `rating ` - Rating the product has got from its consumers
*  `description ` - Description of the dataset (in detail)
